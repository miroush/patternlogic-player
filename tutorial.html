1. Úvod — co to je a pro koho

patternlogic-player je JavaScriptová knihovna, která ti umožní přehrávat kytarové patterny (rozepsané jako JSON) s reálnými samply nebo synťákem. Postavená na Tone.js v14.

Pro koho: pro aplikace, které chtějí hrát kytarové doprovody nad posloupností akordů. PatternLogic (knihovna patternů + editor) a Vibechords (chord progression player) ji obě používají. Knihovna je univerzální — umí jakýkoliv Tone.Sampler.

Co knihovna řeší za tebe

  • Voicing engine — spočítá, jaké noty zahrát na kterých strunách pro daný akord v daném tvaru (E-barre / A-barre / oktáva výš).
  • Pattern scheduler — přehraje pattern (16 nebo 32 kroků) v zadaném tempu, s humanizací (rozkmit velocity, timing, strum delay).
  • Sequence scheduler — přehraje celý chord progression (C G Am F) a zvládá i mix 16n a 32n patternů v jedné sekvenci.
  • Hot-update — za běhu ti přepne pattern, tempo, taktový podpis nebo i celý nástroj bez stopu přehrávání.
  • Eventy — 14 eventů (step, note, loop, chord, …), na které si v hostovské appce navěsíš UI glue (zvýrazňování, highlight, metronom).

Co knihovna NEŘEŠÍ

  • UI — fretboard, step indikátor, editor patternu. To je práce hostující aplikace.
  • Basa — zatím zůstává v hostu (PatternLogic má inline bass engine). Migrace do knihovny je plánovaná.
  • Efektový řetězec — EQ, reverb, cab IR. Nasypeš samplerem, kam chceš (už prošlou přes efekty).
  • Sampling — samotné zvukové samply si přineseš jako Tone.Sampler. Knihovna je agnostická.

2. Quick start — první zvuk za 15 řádků

Minimální kód, který hraje C dur pattern. Místo reálného sampleru použijeme Tone.PolySynth (stejné API, žádný download samplů).

// 1. Importuj knihovnu (ES module)
import { PatternPlayer } from './pattern-player.js';

// 2. Udělej "sampler" — synth jako náhrada (má stejné API jako Tone.Sampler)
const synth = new Tone.PolySynth(Tone.Synth).toDestination();
await Tone.start(); // vyžaduje klik uživatele

// 3. Vytvoř playera a pattern
const player = new PatternPlayer({ guitarSampler: synth, bpm: 100 });
const pattern = {
  id: 'hello',
  loop_steps: 16,
  resolution: '16n',
  meta: { style: 'strum' },
  steps: [
    { beat: '1', strings: ['bass_root', 4, 3, 2, 1], direction: 'D', duration: '4n' },
    { beat: '3', strings: ['bass_alt',  4, 3, 2, 1], direction: 'D', duration: '4n' }
  ]
};

// 4. Spusť
player.setChord({ root: 'C', quality: 'maj' });
player.loadPattern(pattern);
player.play();
Živá ukázka
Co se právě stalo: knihovna si z akordu C:maj spočítala E-tvar na 8. pražci (C dur bez otevřených strun), voicing pro každou strunu a pattern scheduler začal triggerovat tóny na dobu 1 a 3. Strum delay (20 ms) a ring-until-next-hit (nota drží, dokud neudeří další) jsou zapnuté by default.

3. Voicing engine — pure utilities

Pod kapotou knihovny jsou čisté matematické funkce, které ti přepočítají abstraktní popis akordu na konkrétní noty. Nepotřebují audio context — můžeš je volat i bez Tone.js, jsou to jen výpočty.

getAllPositions — všechny hratelné pozice akordu

import { getAllPositions } from './pattern-player.js';

const positions = getAllPositions({ root: 'C', quality: 'maj' });
// [
//   { shape: 'E', barFret: 8,  baseNote: 'C3', ... },
//   { shape: 'A', barFret: 3,  baseNote: 'C3', ... },
//   { shape: 'E', barFret: 20, ... }  // pokud vejde (≤ 15. pražec default)
// ]

Každá pozice má shape (E/A), barFret (kde leží barre), baseNote (nota na bass struně) a další metadata. Aplikace může uživateli nabídnout výběr pozice (PatternLogic kreslí všechny na hmatník a nechá kliknout).

generateVoicing — noty pro všech 6 strun

import { generateVoicing } from './pattern-player.js';

const pos = positions[0]; // E-tvar, 8. pražec
const voicing = generateVoicing(pos, 'maj');
// {
//   1: 'C5', 2: 'G4', 3: 'E4', 4: 'C4', 5: 'G3', 6: 'C3',
//   bass_root: 'C3', bass_alt: 'G3'
// }

Klíče 1–6 jsou fyzické struny (1 = vysoké e). bass_root a bass_alt jsou abstraktní reference (hodí se pro basové patterny, které nevědí, jestli hrajou E nebo A tvar).

resolveStrings — abstraktní → fyzické struny

import { resolveStrings } from './pattern-player.js';

resolveStrings(['bass_root', 4, 3], 'E'); // [6, 4, 3]
resolveStrings(['bass_root', 4, 3], 'A'); // [5, 4, 3]

(výstup)
Proč pure utilities? Můžeš je použít i v contextu, kde zvuk nehraješ — třeba v generátoru notových materiálů, MIDI exportu, nebo jen pro vizualizaci hmatníku.

4. Single-chord přehrávání

Typický use case: uživatel má vybraný jeden akord a loopuje jeden pattern dokola (kytarový editor, metronom pro cvičení).

Konstruktor + základní ovládání

const player = new PatternPlayer({
  guitarSampler: mySampler,        // REQUIRED — Tone.Sampler instance
  muteSampler: myMuteSampler,      // optional — palm-mute varianta
  bpm: 100,
  timeSignature: '4/4',            // '4/4' nebo '3/4'
  humanization: {
    velVariance: 0.08,            // rozkmit velocity 0–1
    accentStrength: 0.15,
    timingVarianceMs: 5             // rozkmit timingu ± ms
  }
});

player.setChord({ root: 'G', quality: 'maj', shape: 'auto', shapeOctave: 1 });
player.loadPattern(myPattern);
player.play();
// … user kliká na různé akordy …
player.setChord({ root: 'D', quality: 'min' }); // přepne za běhu
player.stop();

Eventy, které ti přijdou

  • chord — při každém setChord(). Payload: { root, quality, shape, barFret, voicing, position }.
  • pattern — při loadPattern nebo hotUpdatePattern. Payload: { loopLen, resolution, pattern }.
  • start, stop, loop — hranice přehrávání.
  • step — každý tick scheduleru. Payload: { step, loopLen, time, resolution }. Používá se na step indikátor a metronom.
  • note — každý trigger noty. Payload: { notes, vel, dur, strings, technique, direction, time, event }. Používá se na highlight strun.
  • error — při neplatné notě nebo jiné chybě.
Živá ukázka — single-chord s eventy

Strum pattern použitý v demu: klasický 4-úderový strum (D — D U — D — D U) přes 16 kroků, 4n ring, strum delay 20 ms. Když změníš akord za běhu, knihovna přepne voicing hned (na dalším triggeru).

5. Sekvence akordů

Druhý režim: uživatel zapíše posloupnost akordů jako text a knihovna je přehraje jeden za druhým. Pro chord progressions (C G Am F), songy, cvičení.

Notace sekvence

ZápisVýznam
AA dur, celý takt (4 doby ve 4/4)
Ama moll, celý takt
Amin7, Gsus4kvalita za rootem
C.C dur, půl taktu (2 doby)
G..G dur, 1 doba
G...G dur, 1/2 doby
Am(001)A moll, pattern s kódem 001
F(e) / F(a)F dur v E-tvaru / A-tvaru
G(a2)G dur v A-tvaru o oktávu výš
C.(abc e)C dur, půl taktu, pattern abc, E-tvar

playSequence API

player.playSequence('C G Am F', {
  defaultPattern: myPattern,                // REQUIRED (nebo getDefaultPattern)
  loop: true,                             // loopovat celou sekvenci (default true)
  timeSignature: '4/4',                   // '4/4' nebo '3/4'
  patNumMap: { '001': 'pattern-uuid-1' }, // kód → id pro lookup
  getPattern: (id) => PATTERNS.find(p => p.id === id),
  getText: () => textarea.value,            // živě reparse textu na chord boundary
  getDefaultPattern: () => editorGetPattern() // živě recompile patternu na loop
});

Callbacky pro živé změny

  • getText() — knihovna na hranici každého akordu volá a porovná s minulým textem. Pokud se liší, reparsuje a od následujícího akordu hraje podle nové verze. Uživatel píše do textarey a změny se projeví postupně.
  • getDefaultPattern() — na hranici každého loopu volá a přenačte pattern. Uživatel edituje v editoru a změny se projeví od dalšího taktu.

Eventy specifické pro sekvenci

  • seq-start — payload: { chords, loop }.
  • seq-chord — přechod na další akord. Payload: { chord, index }.
  • seq-reparse — text se změnil. Payload: { chords, reason? }.
  • seq-loop — sekvence se vrátila na první akord.
  • step, note, loop, pattern mají navíc sequence: true ve payloadu, abys v handleru odlišil.
Živá ukázka — sekvence s live reparse

Tip: během přehrávání přepiš textareu na E A D G — změna se projeví na hranici dalšího akordu.

6. Hot-update za běhu

Knihovna umí přepínat parametry bez stopnutí Transportu a restartu scheduleru. Využitelné v aplikacích, kde user hýbe slidery nebo edituje pattern a chce slyšet změny okamžitě.

hotUpdatePattern(newPattern)

Přepne aktuálně hrající pattern za jiný. Zachová si pozici (u single-chord modulo přes nový loopLen, u sekvence se přepne na dalším loop boundary). Pokud se změní resolution (16n ↔ 32n), scheduler se automaticky překreslí.

player.hotUpdatePattern(newPat);
// Event 'pattern' s hot: true ti přijde, ať víš, že to byl hot swap

setBpm(n)

Změní tempo. Transport se přeladí okamžitě, strum delay se přeškáluje podle edit_bpm / playback_bpm.

setTimeSignature(ts)

'4/4' nebo '3/4'. Pokud právě hraje sekvence, reparsuje se aktuální text s novým podpisem (v 3/4 má akord bez teček 3 doby místo 4). Emituje seq-reparse s reason: 'timeSignature'.

setGuitarSampler / setMuteSampler

Výměna nástroje za běhu. Knihovna od dalšího triggeru volá nový sampler. Doznívající noty na starém sampleru dohrají — pokud chceš tvrdé cutoff, zavolej oldSampler.releaseAll().

7. Eventy — kompletní reference

Kompletní seznam eventů, které emituje PatternPlayer. Registrace přes player.on(name, handler), odregistrace player.off(name, handler?) (bez handleru smaže všechny).

EventKdy se emitujePayload
chord Po každém setChord() (i implicitním na začátku sekvence) { root, quality, shape, barFret, voicing, position }
pattern loadPattern nebo hotUpdatePattern { loopLen, resolution, pattern, hot?: true, sequence?: true }
start Po play() nebo playSequence() { time, sequence?: true }
stop Po stop() nebo po doběhnutí sekvence bez loopu { sequence }
step Každý tick scheduleru (16n nebo 32n dle patternu) { step, loopLen, time, resolution, sequence?, chordIndex? }
note Každý trigger noty (včetně strun v jednom strumu) { notes, vel, dur, strings, technique, direction, time, event, sequence? }
loop Pattern/sekvence se vrátila na začátek { time }
bpm Po setBpm() { bpm }
error Chyba v triggeru (sampler selhal, neplatná nota) { err, note?, event?, where? }
tick Granulární tick v sekvenci (32n grid) { rawTick, time }
seq-start Po playSequence() { chords, loop }
seq-chord Přechod na další akord sekvence { chord, index }
seq-reparse Text se změnil nebo setTimeSignature za běhu { chords, reason?, timeSignature? }
seq-loop Sekvence se vrátila na první akord { time }

8. Integrační sketch pro editor

Typický use case v hostující aplikaci (PatternLogic, Vibechords): editor patternu + sekvence textarea + hot-update bez stopnutí přehrávání. Tady je minimální integrace.

// Stát hostu
let player = null;
const PATTERNS = [/* pole JSON patternů */];
const patNumMap = { '001': 'my-strum', '002': 'my-fp' };

// Helpery ke čtení UI
function editorGetPattern() {
  // … vybere rows z editoru, vrátí JSON pattern
  return { /* ... */ };
}

// Start sekvence
document.getElementById('play-seq').addEventListener('click', async () => {
  await Tone.start();
  if (!player) {
    player = new PatternPlayer({ guitarSampler: mySampler, bpm: 100 });
    player.on('seq-chord', ({ chord, index }) => highlightChordChip(index));
    player.on('step', ({ step }) => updateStepIndicator(step));
    player.on('stop', () => resetUI());
  }

  player.playSequence(document.getElementById('seq-input').value, {
    defaultPattern: editorGetPattern(),
    getPattern: (id) => PATTERNS.find(p => p.id === id),
    getText: () => document.getElementById('seq-input').value,
    getDefaultPattern: () => editorGetPattern(),
    patNumMap,
    loop: true,
  });
});

// Hot-update z Apply tlačítka v editoru
document.getElementById('apply').addEventListener('click', () => {
  if (player && player._isPlaying) {
    player.hotUpdatePattern(editorGetPattern());
  }
});

// BPM slider
document.getElementById('bpm').addEventListener('input', (e) => {
  player?.setBpm(+e.target.value);
});
Klíč je oddělení: knihovna si řeší scheduler, voicing, timing, ringy. Hostující aplikace jen dodává data (pattern, text sekvence, sampler) a přes callbacky reaguje na eventy (highlight, stav tlačítek). Hosta neřídí knihovna, knihovna čte hosta přes callbacky.

9. Co zatím chybí a co přijde

Aktuálně v knihovně NEJSOU

  • Bass engine — basový nástroj s vlastními basovými patterny (pattern s instrument: 'bass', triggery přes Tone.MonoSynth nebo bass sampler). V PatternLogic zůstává jako paralelní inline scheduleRepeat. Plánovaná migrace do knihovny: jedna vlna, výsledek bude unifikovaný scheduler pro kytaru + basu v jedné instanci PatternPlayer.
  • Piano / drums engines — formát patternu to umí (meta.instrument: 'piano'), ale voicing engine pro klávesy není implementovaný. Piano patterny neřeší E/A tvar, potřebují vlastní voicingová pravidla.
  • Multi-track mix — hrát víc patternů nad jedním akordem současně (kytara + basa + perkuse). V plánu.

Kam se knihovna vyvíjí

  • Bass engine (priorita 1 — odbourá dnešní duplicitu v PatternLogic hostu)
  • Uvolnění stabilní 1.0 po dokončení bass migrace + real-world usage ve Vibechords
  • Piano engine (priorita 2, na vyžádání)
